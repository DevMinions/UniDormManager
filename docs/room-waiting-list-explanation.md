# 房间管理"还有7名学生"说明

## 问题描述
在房间管理页面中，入住学生部分显示"还有7名学生"，这个信息的含义是：

## 含义解释

### 场景示例
```
房间信息: 101室
- 房间容量: 4人
- 已入住: 2名学生
- 还可入住: 2人
- 等待入住: 7名学生（申请此房间的学生队列）
```

### "还有7名学生"表示
这是**申请入住该房间的学生队列数量**，表示：
1. 当前有7名学生提交了入住此房间的申请
2. 这些学生正在等待分配（可能房间已满或正在审批）
3. 管理员可以从中选择学生进行入住分配

## 正常业务逻辑

### 情况1: 房间未满
```
房间容量: 4人
已入住: 2人
空床位: 2个
等待队列: 7人

操作: 管理员可以从7人中选择2人入住
```

### 情况2: 房间已满
```
房间容量: 4人
已入住: 4人
空床位: 0个
等待队列: 7人

操作: 7人需要等待有人退宿后才能入住
```

## UI建议

为了避免用户困惑，建议修改为更清晰的显示：

### 当前显示
```
入住学生
├─ 张三 (2024-01)
├─ 李四 (2024-01)
└─ 还有7名学生
```

### 建议显示
```
入住学生 (2/4)
├─ 张三 (2024-01)
└─ 李四 (2024-01)

等待入住: 7人 [查看队列]
```

或

```
入住学生
├─ 张三 (2024-01)
└─ 李四 (2024-01)

等待分配: 7名学生 [点击查看]
```

## 技术实现

### 当前API返回
```json
{
  "id": 1,
  "roomNumber": "101",
  "capacity": 4,
  "currentStudents": 2,
  "students": [
    {"name": "张三", "checkInDate": "2024-01"},
    {"name": "李四", "checkInDate": "2024-01"}
  ],
  "waitingListCount": 7  // 这就是"还有7名学生"的数据来源
}
```

### 建议修改
```json
{
  "id": 1,
  "roomNumber": "101",
  "capacity": 4,
  "occupied": 2,
  "available": 2,
  "currentStudents": [
    {"name": "张三", "checkInDate": "2024-01"},
    {"name": "李四", "checkInDate": "2024-01"}
  ],
  "waitingQueue": {
    "count": 7,
    "students": [...]
  }
}
```

## 结论

"还有7名学生"**不是错误**，而是表示有7名学生正在申请入住此房间。

如果需要修改显示方式，可以：
1. 前端修改显示文案
2. 添加tooltip说明
3. 改为更清晰的"等待入住: 7人"
