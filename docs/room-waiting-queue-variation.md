# 房间"还有N名学生"数量差异说明

## 你的观察
你发现不同房间显示的等待学生数量不同：
- 房间A: "还有7名学生"
- 房间B: "还有5名学生"
- 房间C: "还有3名学生"

## 为什么数量不同？

### 1. 这不是错误，是正常现象
每个房间的等待队列是**独立的**，数量不同说明：
- 不同房间的受欢迎程度不同
- 学生偏好不同（如：靠近食堂、安静、朝阳等）
- 有些房间因为设施好、位置佳，申请人数更多

### 2. 与性别分配的关系

你说得对，**性别是重要因素之一**：

```
男生宿舍区
├── 101室（4人间）: "还有7名学生"等待 ← 申请人数多
├── 102室（4人间）: "还有3名学生"等待 ← 申请人数少
└── 103室（2人间）: "还有5名学生"等待

女生宿舍区
├── 201室（4人间）: "还有8名学生"等待 ← 更受欢迎
├── 202室（4人间）: "还有2名学生"等待
└── 203室（6人间）: "还有4名学生"等待
```

### 3. 影响等待人数的因素

| 因素 | 影响 |
|------|------|
| **性别** | 男女分宿，各自独立排队 |
| **房间类型** | 2人间比6人间更抢手 |
| **楼层** | 低楼层（方便）vs 高楼层（安静）|
| **位置** | 靠近食堂/澡堂/教学楼的房间更受欢迎 |
| **设施** | 有空调、独立卫浴的房间申请人数多 |
| **价格** | 便宜的房间申请人数通常更多 |
| **历史口碑** | 学长学姐推荐会影响选择 |

### 4. 业务逻辑示例

```
场景：新学期开学，学生申请宿舍

男生宿舍楼A（靠近食堂，设施新）
├─ 101室（4人间，有空调）: 20人申请 → 显示"还有16名学生"
├─ 102室（4人间，无空调）: 5人申请 → 显示"还有1名学生"
└─ 103室（6人间，有空调）: 12人申请 → 显示"还有6名学生"

男生宿舍楼B（偏远，设施旧）
├─ 201室（4人间）: 3人申请 → 显示"还有0名学生"
├─ 202室（4人间）: 2人申请 → 显示"还有0名学生"
└─ 203室（6人间）: 1人申请 → 显示"还有0名学生"
```

## 结论

**数量差异是正常的业务现象**，反映的是：
1. 不同房间的供需关系
2. 学生的偏好和选择
3. 房间的客观条件差异

**这与性别分配有关，但不完全是性别决定的**，楼层、设施、位置等因素同样重要。

如果你看到某个房间显示"还有50名学生"，那说明这个房间非常抢手；
如果显示"还有0名学生"，说明这个房间可能条件较差或刚有人退宿。
