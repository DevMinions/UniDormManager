## 数据模型对比

### User 类型定义

**需求文档定义**：
```typescript
interface User {
  id: string;
  username: string;
  email: string;
  phone: string;
  realName: string;
  status: 'Active' | 'Inactive' | 'Suspended';
  lastLoginAt?: string;
  createdAt: string;
  updatedAt: string;
  roles?: Role[];
  studentId?: string;
}
```

**实际实现** (`types.ts`):
```typescript
interface User {
  id: string;
  username: string;
  email: string;
  phone: string;
  realName: string;
  status: 'Active' | 'Inactive' | 'Suspended';
  lastLoginAt?: string;
  createdAt: string;
  roleIds: string[];  // ✅ 使用 ID 数组而非完整 Role 对象
  avatar?: string;    // ⚠️ 额外字段
}
```

**差异**：
- ✅ 基本字段符合
- ⚠️ `roles?: Role[]` vs `roleIds: string[]` - 实现使用 ID 数组（更合理）
- ⚠️ 缺少 `updatedAt`
- ⚠️ 缺少 `studentId?`
- ➕ 额外有 `avatar?`

### Role 类型定义

**需求文档定义**：
```typescript
interface Role {
  id: string;
  code: string;
  name: string;
  description: string;
  level: number;
  permissions?: Permission[];
  createdAt: string;
  updatedAt: string;
}
```

**实际实现** (`types.ts`):
```typescript
interface Role {
  id: string;
  code: string;
  name: string;
  description: string;
  level: number;
  permissionIds: string[];  // ✅ 使用 ID 数组
  isSystem?: boolean;       // ➕ 额外字段，标识预定义角色
}
```

**差异**：
- ✅ 基本字段符合
- ⚠️ `permissions?: Permission[]` vs `permissionIds: string[]` - 实现使用 ID 数组（更合理）
- ⚠️ 缺少 `createdAt` 和 `updatedAt`
- ➕ 额外有 `isSystem?`（用于保护预定义角色）

### Permission 类型定义

**需求文档定义** vs **实际实现**：✅ 完全符合

2. **缺少页面级权限检查**

   - 现状：只在侧边栏有权限检查，页面内按钮未做权限控制
   - 建议：在每个操作按钮前检查权限，无权限时隐藏或禁用

3. **用户管理页面功能不完整**
   - 缺少状态筛选
   - 缺少角色筛选
   - 缺少关联学生功能
   - 缺少楼栋选择（楼栋管理员角色）

### 🟡 中优先级问题

1. **角色管理页面**
   - 缺少权限范围（Scope）配置

2. **权限列表页面**
   - 缺少权限详情展开

3. **分页功能**

### 🟢 低优先级优化
