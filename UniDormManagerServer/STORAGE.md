# 存储和缓存架构说明

## 当前状态

✅ **已完成数据持久化**
- PostgreSQL 数据库支持
- Redis 缓存支持
- 自动数据库迁移
- 缓存失效策略

## 架构设计

### 存储层架构

```
┌─────────────────┐
│   HTTP Handlers │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  StoreInterface │  ← 统一接口
└────────┬────────┘
         │
         ▼
┌──────────┐
│ Database │
│  Store   │
└────┬─────┘
     │
     ▼
┌──────────┐
│PostgreSQL│ ← 数据持久化（必需）
└──────────┘
     │
     ▼
┌──────────┐
│  Redis   │ ← 缓存层（可选）
└──────────┘
```

### 存储实现

1. **数据库存储** (`store/store_db.go`)
   - 使用 PostgreSQL 持久化数据
   - 支持事务和索引
   - 自动创建表结构
   - **必需**：数据库连接失败时服务器无法启动

2. **缓存层** (`cache/cache.go`)
   - Redis 缓存热点数据
   - 自动缓存失效
   - 减少数据库查询
   - **可选**：缓存失败不影响功能

## 使用方式

**注意**: 本项目现在**必须**使用 PostgreSQL 数据库。数据库连接失败时服务器将无法启动。

### 1. 数据库存储（必需）

```bash
# 设置环境变量
export DB_HOST=localhost
export DB_PORT=5432
export DB_USER=postgres
export DB_PASSWORD=postgres
export DB_NAME=unidorm
export DB_SSLMODE=disable

go run main.go
```

### 2. 数据库 + Redis 缓存（推荐）

```bash
# 设置环境变量
export USE_CACHE=true
export DB_HOST=localhost
export DB_PORT=5432
export DB_USER=postgres
export DB_PASSWORD=postgres
export DB_NAME=unidorm
export DB_SSLMODE=disable
export REDIS_HOST=localhost
export REDIS_PORT=6379

go run main.go
```

## 缓存策略

### 缓存键设计

- `student:{id}` - 单个学生
- `students:all` - 所有学生列表
- `building:{id}` - 单个楼栋
- `buildings:all` - 所有楼栋列表
- `room:{id}` - 单个房间
- `rooms:all` - 所有房间列表
- `repair:{id}` - 单个报修请求
- `repairs:all` - 所有报修列表
- `notice:{id}` - 单个公告
- `notices:all` - 所有公告列表
- `dashboard:stats` - 仪表板统计

### 缓存过期时间

| 数据类型 | 过期时间 | 原因 |
|---------|---------|------|
| 房间列表 | 5分钟 | 变更频繁 |
| 报修列表 | 5分钟 | 变更频繁 |
| 仪表板统计 | 5分钟 | 实时性要求高 |
| 学生列表 | 15分钟 | 变更中等 |
| 公告列表 | 15分钟 | 变更中等 |
| 楼栋列表 | 1小时 | 变更频率低 |

### 缓存失效

当数据发生变更时，自动清除相关缓存：

- **创建操作**: 清除列表缓存
- **更新操作**: 清除列表缓存 + 单个资源缓存
- **删除操作**: 清除列表缓存 + 单个资源缓存
- **仪表板相关**: 房间、报修变更时清除仪表板缓存

## 性能优化

### 数据库优化

1. **索引优化**
   - 学生表：`student_id`、`room_number` 索引
   - 房间表：`building` 索引
   - 报修表：`status`、`room_number` 索引
   - 公告表：`date` 索引

2. **连接池**
   - 使用 `pgxpool` 管理连接
   - 自动处理连接复用

3. **查询优化**
   - 使用参数化查询防止 SQL 注入
   - 避免 N+1 查询问题

### 缓存优化

1. **缓存命中率**
   - 频繁查询的数据优先缓存
   - 根据访问模式调整缓存策略

2. **缓存预热**
   - 服务器启动时可选择性预热热点数据

3. **缓存穿透防护**
   - 对不存在的键也进行短时间缓存
   - 避免频繁查询数据库

## 数据一致性

### 缓存一致性策略

1. **写后删除** (Write-Through)
   - 更新数据库后立即删除缓存
   - 下次查询时重新加载

2. **延迟双删** (Double Delete)
   - 更新前删除缓存
   - 更新后再次删除缓存（防止并发问题）

3. **版本控制**
   - 为缓存添加版本号
   - 版本不匹配时自动失效

## 故障处理

### 数据库故障

- **服务器无法启动**：数据库连接失败时，服务器会直接退出
- 记录详细错误日志
- 健康检查端点显示数据库状态
- 请确保 PostgreSQL 正在运行且配置正确

### Redis 故障

- 继续使用数据库（缓存失败不影响功能）
- 记录警告日志
- 健康检查端点显示缓存状态
