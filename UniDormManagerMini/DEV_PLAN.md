# 大学宿舍管理系统 - 微信小程序版本开发计划

## 📋 项目概述

**项目名称**: UniDorm Manager Mini（大学宿舍管理系统小程序版）
**目标用户**: 宿舍管理员、学生
**核心功能**: 提供移动端的宿舍管理服务，包括报修、公告、查询等功能

---

## 🎯 功能需求

### 核心功能模块

#### 1️⃣ 用户认证模块
- 微信登录
- 用户身份识别（管理员/学生）
- 权限管理

#### 2️⃣ 仪表板模块
- 统计数据展示（房间总数、空闲房间、报修数量等）
- 快捷入口

#### 3️⃣ 房间查询模块
- 按楼栋查询房间
- 按房间号搜索
- 房间详情展示
- 房间占用状态

#### 4️⃣ 报修管理模块（学生端）
- 提交报修申请
- 查看报修进度
- 报修历史记录
- 图片上传

#### 5️⃣ 报修处理模块（管理员端）
- 查看所有报修请求
- 更新报修状态
- 处理优先级排序
- 统计报修数据

#### 6️⃣ 公告通知模块
- 查看最新公告
- 公告分类显示
- 公告详情

#### 7️⃣ 楼栋管理模块（管理员端）
- 楼栋列表展示
- 楼栋详情查看

---

## 🏗️ 技术栈

- **小程序框架**: 微信小程序原生开发
- **开发工具**: 微信开发者工具
- **UI组件**: WeUI 或 Vant Weapp
- **状态管理**: 自定义状态管理
- **网络请求**: wx.request 封装
- **后端接口**: 复用现有 Go 后端 API

## 🔌 API 接口对接

### 基础配置
```
API_BASE_URL: http://your-server:8080
```

### 接口映射
- 登录认证: 自定义接口（需要后端新增）
- 仪表板统计: GET /api/dashboard/stats
- 房间查询: GET /api/rooms
- 报修管理: GET/POST/PUT /api/repairs
- 公告通知: GET /api/notices

## 📱 UI 设计原则

1. **简洁明了**: 避免复杂操作，突出核心功能
2. **符合小程序规范**: 遵循微信小程序设计规范
3. **响应式设计**: 适配不同屏幕尺寸
4. **用户友好**: 清晰的操作指引和反馈

---

## 🔒 安全考虑

1. 用户认证和授权
2. 数据传输加密（HTTPS）
3. 敏感信息保护
4. 权限控制（学生/管理员）

---

## 📊 数据存储

1. **本地存储**: 使用 wx.storage 存储用户信息和token
2. **会话管理**: 维护登录状态
3. **缓存策略**: 合理缓存静态数据
